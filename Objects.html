JavaScript Objects

Explanation: By this point you should already know that variables in javascript hold data. In javascript objects are variables that can should multiple pieces of data. We can compare ojects to people, places, or things in real life. A car has a model, color, and year among other possible properties. We can create an object to store these properties, allowing them to be read from and written to during an the execution of a webpage.

Creation and Properties

You can define and create an object as follows.

var newCar = {model:"Saturn",color:"Blue",year:"2019"};

This would create a variable object named newCar. The words inside the braces are name:value pairs separated by commas, and define what the object newCar contain.

The values contained in each property can be accessed in the following two formats

objectName.propertyName   or
objectName["propertyName"]

Functions and Object Methods

In many languages a function is a chunk of code that can be called upon to execute from elsewhere in a program. JavaScript is no different. In JavaScript Objects can contain methods. Methods are stored as object properties, but instead of simply containing data the property name contains a function. When the Object method is called the actions programmed in its function statement are executed.

Continuing with our newCar variable from above, lets add a method that will allow us to get the newCar's current model. The function would be written as such.

function(){
return this.model;
}

 Or written on one line it would appear as below.

function(){return this.model;}

This would then be used as the "data" to be included in the methodName property.

var newCar = {model:"Saturn",color:"Blue",year:"2019",getModel:function(){return this.model;}};

Of course this can become a very confusing line so separating an Object into multiple lines as follows is quite common.

var newCar = {
   model:"Saturn",
   color:"Blue",
   year:"2019",
   getModel:function(){
     return this.model;
   }
};

Accessing Object Methods An Object method wouldn't be very usful if you couldn't access it. Object methods can be accessed using the objectName.propertyName format from before but with a slight variation. If I were to only use objectName.propertyName I would get the entire function returned as data. If I want the actual method to be executed I need to use two parenthesis. This would appear as objectName.propertyName() and is often used in an equation or when assigned to another variable. I'll demonstrate with our example.

Instantiation

In it's simplest form, to instantiate is to create an instance (or specific occurance) of something. Lets say we want to create a list of people with a first name, hair color, and age. Instead of repeating the same code and changing attribute values, we can use a function that creates the objects for us. Used this way a "Person" function would be comparable to a "class" in other object oriented languages. The function is a blueprint and we use that blueprint to create individual objects. This is one method of instantiating.

To do this we would first create the Person object as follows.

function Person(firstName, hairColor, age) {
  this.firstName = firstName;
  this.hairColor = hairColor;
  this.age = age;
}

Then we could create a few people as follows.

var doctor = new Person("Bill","Black",38);
var designer = new Person("Sarah","Brown", 27);
var tech = new Person("Joe","Blond",41);

Inheritance

Most people are familiar with the the concept that children inherit some qualities from their parents. In javascript, as with most programming languages, there are methods to have "children" objects inherit from "parent" objects. This can be useful when we have a object who's properties and structure should be the same as or very similar to another object.

"When using JavaScript it can be helpful to think of inheritance with a "family" tree. All JavaScript objects inherit properties and methods from a prototype...Date objects inherit from Date.prototype. Array objects inherit from Array.prototype. The Object.prototype is on the top of the prototype inheritance [tree]. Date objects, Array objects, and Person objects inherit from Object.prototype."
